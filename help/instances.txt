awsdo instances - Manage EC2 instances

USAGE:
    awsdo instances find [--profile <aws cli profile>] [--filter <filter text>]
    awsdo instances list [--profile <aws cli profile>]
    awsdo instances ls [--profile <aws cli profile>]
    awsdo instances add [--profile <aws cli profile>] [--name <instance name>] [--filter <filter text>]
    awsdo instances update [--profile <aws cli profile>] [--filter <filter text>] [<instance name>]
    awsdo instances remove [--profile <aws cli profile>] [--name <instance name>]
    awsdo instances rm [--profile <aws cli profile>] [--name <instance name>]

    # Linux-style commands (alternative syntax)
    awsdo ls instances [--profile <aws cli profile>]
    awsdo list instances [--profile <aws cli profile>]
    awsdo ls bastions [--profile <aws cli profile>]
    awsdo list bastions [--profile <aws cli profile>]
    awsdo add instance [--profile <aws cli profile>] [--name <instance name>] [--filter <filter text>]
    awsdo add bastion [--profile <aws cli profile>]
    awsdo rm instance [--profile <aws cli profile>] [--name <instance name>]
    awsdo rm bastion [--profile <aws cli profile>] [--name <bastion name>]
    awsdo update instance [--profile <aws cli profile>] [--filter <filter text>] [<instance name>]
    awsdo find instance [--profile <aws cli profile>] [--filter <filter text>]

DESCRIPTION:
    The instances command provides subcommands to find and manage EC2 instances.
    Instances can be named and stored in configuration for easy access.

SUBCOMMANDS:
    find    Find EC2 instances whose Name tag contains the specified filter string.
            Displays instance ID and name. If exactly one instance is found (and
            it's not a bastion), it is automatically saved as the default instance.

    list    List all configured named instances for the specified profile.
            Displays instance information in a vertical format. The default
            instance (if set) will be marked.
            Shortcut: ls

    add     Add a new named EC2 instance to the configuration. Queries AWS for
            instances matching the filter, allows selection, and prompts for a name.
            The instance name can also be provided via the --name flag.

    update  Refresh a named instance from AWS using a Name-tag filter. The filter
            used when adding or updating is saved in configuration and reused on
            later updates. Profile can be omitted if the instance name is unique
            across configured profiles.

    remove  Remove a named instance from the configuration. Prompts for
            confirmation before removal. The instance name can be provided via
            the --name flag or will be prompted interactively.
            Shortcut: rm

OPTIONS:
    --profile, -p    AWS CLI profile to use
    --name, -n       Instance name (for add and remove commands)
    --filter, -f     Filter text to match against instance Name tags (for find, add,
                     and update commands)

FIND COMMAND:
    Finds EC2 instances whose Name tag contains the specified filter string.
    Displays instance ID and name for each matching instance. If exactly one
    instance is found (and it's not a bastion), it is automatically saved as
    the default instance for the profile.

    The filter can be specified using the --filter or -f flag. If the filter
    is not provided, you will be prompted to enter it interactively.

    Examples:
        awsdo instances find --filter example
        awsdo instances find -f example
        awsdo instances find -p dev -f myapp
        awsdo instances find -p dev    # Will prompt for filter
        awsdo find instance -f example

LIST COMMAND:
    Lists all configured instances for the specified profile in a vertical format,
    showing name, instance ID, profile, and host for each instance. The default
    instance (if set) will be marked with "(default)". If no profile is specified,
    lists instances across all profiles.

ADD COMMAND:
    Provides an interactive interface to add a new named instance:
    1. Queries AWS for EC2 instances matching the filter string
    2. Displays available instances for selection
    3. Prompts for instance name (or uses --name flag if provided)
    4. Saves the instance configuration

    The filter can be specified using the --filter or -f flag. If the filter
    is not provided, you will be prompted to enter it interactively.

    Examples:
        awsdo instances add --filter example
        awsdo instances add -f example -n myinstance
        awsdo instances add -p dev -f myapp
        awsdo instances add -p dev    # Will prompt for filter
        awsdo add instance -f example

UPDATE COMMAND:
    Updates an existing named instance from AWS:
    1. Resolves the AWS profile: use -p if provided; otherwise if exactly one
       configured instance matches the name, use its profile; if several match,
       you choose which profile to use.
    2. Uses --filter or -f if provided; otherwise uses the filter saved in
       configuration from the last add/update; if none is saved, prompts for it.
    3. Queries EC2, displays matches, and saves the selected instance (including
       the filter string used).

    The instance name is a positional argument (not --name). Flags may appear
    before or after the name.

    Examples:
        awsdo instances update myinstance
        awsdo instances update -f myapp myinstance
        awsdo instances update myinstance --filter myapp
        awsdo instances update -p dev myinstance
        awsdo update instance myinstance

REMOVE COMMAND:
    Removes a named instance from the configuration:
    1. Prompts for instance name (or uses --name flag if provided)
    2. Displays instance information
    3. Asks for confirmation before removal
    4. Clears default instance if the removed instance was the default

NOTES:
    - The filter text matches instance Name tags using a "contains" match
    - Both traditional (instances find) and Linux-style (find instance) command
      patterns are supported
    - When using Linux-style commands, you can use either singular (instance) or
      plural (instances) forms
