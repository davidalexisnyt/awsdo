awsdo bastions - Manage bastion hosts

USAGE:
    awsdo bastions [list] [--profile <aws cli profile>]
    awsdo bastions ls [--profile <aws cli profile>]
    awsdo bastions add [--profile <aws cli profile>] [--filter <filter text>]
    awsdo bastions update [--profile <aws cli profile>] [--filter <filter text>] [<bastion name>]
    awsdo bastions up [--profile <aws cli profile>] [--filter <filter text>] [<bastion name>]
    awsdo bastions rename [--profile <aws cli profile>] <old name> <new name>
    awsdo bastions mv [--profile <aws cli profile>] <old name> <new name>
    awsdo bastions remove [--profile <aws cli profile>] [--name <bastion name>]
    awsdo bastions rm [--profile <aws cli profile>] [--name <bastion name>]
    awsdo update bastion [--profile <aws cli profile>] [--filter <filter text>] [<bastion name>]
    awsdo rename bastion [--profile <aws cli profile>] <old name> <new name>
    awsdo mv bastion [--profile <aws cli profile>] <old name> <new name>

DESCRIPTION:
    The bastions command provides subcommands to list, add, update, and remove
    bastion hosts. If no subcommand is provided, defaults to 'list'.

SUBCOMMANDS:
    list    List all configured bastions for the specified profile.
            Displays bastion information in a vertical format.
            Shortcut: ls

    add     Interactively configure a new bastion by querying AWS for RDS
            databases and EC2 instances whose Name tag contains the filter
            substring (same as instances add). The filter is saved for later
            updates.

    update  Update an existing bastion configuration. Bastion name is a
            positional argument (or prompted). Profile can be omitted if the
            name is unique across configured profiles. Uses --filter, the saved
            filter, or a prompt. Same interactive flow as add after the filter
            step. Shortcut: up

    rename  Rename a configured bastion. Takes <old name> and <new name> as
            positional arguments (or prompts for them). Profile can be omitted
            if the bastion name is unique across configured profiles. The
            DefaultBastion pointer and bastion ID lookup are updated
            automatically. Shortcut: mv

    remove  Remove a named bastion from the configuration. Prompts for
            confirmation before removal. The bastion name can be provided
            via --name or -n flag, or will be prompted interactively.
            Shortcut: rm

OPTIONS:
    --profile, -p    AWS CLI profile to use
    --filter, -f     Substring for EC2 Name tag when listing bastion instances
                     (for add and update commands)
    --name, -n       Bastion name (for remove only; update uses a positional name)

LIST COMMAND:
    Lists all configured bastions for the specified profile in a vertical
    format, showing name, instance ID, host, port, and local port for each
    bastion. The default bastion (if set) will be marked with "(default)".
    If no profile is specified, lists bastions across all profiles.

ADD COMMAND:
    Provides an interactive interface to configure new bastions. The tool will:
    1. Prompt for filter text (or use --filter / -f) to find EC2 bastion hosts
    2. Query and display available RDS databases
    3. Query and display available bastion EC2 instances matching the filter
    4. Allow you to select a database and bastion instance
    5. Auto-generate a bastion name or prompt for one
    6. Auto-find an available local port
    7. Save the configuration (including the filter)

UPDATE COMMAND:
    Updates an existing bastion configuration. The bastion's ID and profile
    association are preserved during updates. The command will:
    1. Resolve profile from -p, or from the single matching configured bastion
       name, or prompt if multiple profiles have the same name
    2. Use --filter / -f, or the saved filter from configuration, or prompt
       (default suggestion: bastion)
    3. Guide you through the same interactive process as adding a new bastion
    4. Update the configuration with your selections (including the filter)

    Examples:
        awsdo bastions update mybastion
        awsdo bastions update -f bastion mybastion
        awsdo bastions update -p dev mybastion
        awsdo update bastion mybastion

RENAME COMMAND:
    Renames an existing bastion:
    1. Resolves the AWS profile: use -p if provided; otherwise if exactly one
       configured bastion matches the old name, use its profile; if several
       match, you choose which profile to use.
    2. Verifies the new name does not already exist in the target profile.
    3. Updates the bastion entry under the new name, refreshes the bastion ID
       lookup map, and updates the default bastion pointer if applicable.

    Examples:
        awsdo bastions rename oldname newname
        awsdo bastions mv oldname newname
        awsdo bastions rename -p dev oldname newname
        awsdo rename bastion oldname newname
        awsdo mv bastion oldname newname

REMOVE COMMAND:
    Removes a bastion configuration from the specified profile. The command will:
    1. Prompt for bastion name if not provided via --name or -n flag
    2. Display the bastion information
    3. Ask for confirmation before removal
    4. Clear the default bastion if the removed bastion was the default
